秋招阶段性总结

2021-09-24

今天接到了自己秋招的第一个意向，欣喜之余也感到了一丝轻松 —— 压力终于小了一些，可以做一些自己一直以来想做的事情了。前段时间回顾了自己大学以来的学习，似乎都是以考试 / 升学 / 面试为目的，这样显然有诸多不好的地方，其中最为明显的一点就是遗忘的非常快，基本上过了要用的时间就忘了，很难做到积累；现在做 Deep Learning 的论文，就非常后悔之前高数的基础没有打牢。

搭建这个博客的目的其实也比较单纯，一是形成知识的输出，加强自己的理解与记忆；二是了解一下前后端开发上的一些基础知识，毕竟一个算法工程师却不懂工程是有一些说不过去的。初步的话就直接使用了 github 的免费私人站点，搭配 hexo 可以说是方便至极。后续应该会尝试自己去走通搭建 + 部署这一条路，想想本科时候就有很多同学会这些了，实在是有一些羞愧。

秋招目前是接近尾声了，这篇博客就简单记录一下自己秋招的进程以及一些经验吧。

# 面试经验

十几场面试走下来，基本上面试的流程可以分为以下几个部分：

  1. 自我介绍；
  2. 项目 / 论文介绍；
  3. 算法题；
  4. 深度学习、机器学习相关基础知识；
  5. 智力题、概率题（很少）。

一个一个来说吧。

首先是自我介绍。自我介绍其实并不算是很重要的一环，但可以让面试官对你有一个初步的判断，我的建议是先针对岗位对自己有个大概的定义，例如我一般都会说，“我研究生期间所做的工作主要与 NLP 比较相关，用到了很多 NLP 的技术与思想”，主要是让面试官有一种你与岗位较为匹配的感觉。其次，在介绍的过程中可以适当地引导面试官去问你比较擅长解说的项目，比如说我在提到自己的科研经历的时候会稍显随意地说一句 “这个论文还是很有意思的，等会可以展开说一下”，其实就是因为这篇是正在做所以比较熟悉不怕被问哈哈。

然后是项目 / 论文的介绍，这个没啥好说的，全看你自己对细节的掌握程度了。不过有一点要注意的事，但凡是你觉得没有完全熟练掌握的知识点，都不要写到简历上去，避免为自己挖坑。我在字节提前批二面的时候就坑到了自己，我以为简历上写在很下面而且只是一笔带过的 lightGBM 不会被问，结果就被问到了，当时的那个后悔呀。写在简历上一定要全方位无懈可击，不管面试官问啥你都能侃侃而谈。

算法题的话，我的感觉是面试的时候基本没有遇到什么比较难的算法，除了在百度一面的时候做了一道边界条件比较恶心的 DP，其余的基本都是从 leetcode 的 Top100 以及剑指 offer 中可以找到对应原题的。这也没啥好说的，刷就完事的，剑指 offer 一定要在面试前一个月左右再刷一遍。

基础知识的话，可以找一个对应领域学习的 roadmap 来一点点查漏补缺。有一个血的教训，就是一定不要轻易地相信任何知乎 / CSDN / 博客上面写的总结，大概率作者本身（除非比较有名，例如苏神）也是一知半解，甚至带入了自己的误解。对于任何知识点，尽可能地去找源头，例如网上讲解 FM 的文章多如牛毛，但大多数都没讲到点子上，不如去 google scholar 上把 FM 的原论文翻出来看一遍来的实在。印象最深的就是，很多博客都说 GBDT 的残差就是 “目标值与上一棵树预测值之间的差”，而实际上这只是取 MSE 的特例，更为正确的说法应该是损失函数的梯度（所以叫 Gradient boosting）。也不光是面试吧，我以前比较喜欢看别人总结的东西，现在才发现还是自己总结的最靠谱。

最后是智力题、概率题，直接牛客、leetcode、知乎、百度所有能收集到的题目刷一遍就好。

# 写在最后

如果你目前离秋招还有一段时间的话，一定要注意安排好自己的事件。我的秋招几乎与赶 AAAI 论文同步进行，同期还在实习，这就导致了我精力完全不够分配，前期的很多面试都由于准备不充分而或多或少存在一些问题。一个阶段做一个阶段的事情吧，如果我的论文在 7 月之前就搞定了，那今年的 7、8 月份我也就不至于这么痛苦，也就不至于到现在只有 1 个 offer 了。好在是暂时有个归宿了，后面再继续加油。
